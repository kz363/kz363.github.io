Technical Blog #3

April 5, 2014 by Kevin

In the Ruby language, there's always a seemingly large number of ways to complete a certain task. This is how I feel when I look at #cycle. It takes an an enumerable and calls the following code block repeatedly for an n number of times. Here's the example code given in the Ruby documentation:

a = ["a", "b", "c"]
a.cycle { |x| puts x } # print, a, b, c, a, b, c,.. forever.
a.cycle(2) { |x| puts x } # print, a, b, c, a, b, c.

As you can see, it's similar to #times except it's specifically for each element of the enumerable. I'll admit, I had trouble coming up with a viable use for #cycle, but here are some examples that will hopefully help you learn more about it!

The following code will print out a times table for the numbers 1 through 10. #cycle is one of the many methods you could do to iterate through the range (1..10).

i = 1
until i == 11
  (1..10).cycle(1){|x| print "#{x*i} "}
  puts "\n"
  i+=1
end

#cycle takes a block, but if no block is given, it returns an enumerator. This means you can create an "infinite" iteration of sorts. The following is what happens when you try this in IRB:

>> infinite_numbers = (1..3).cycle
=> #<Enumerator: 1..3:cycle>

If you were to keep calling infinite_numbers.next, it'll keep looping from 1, 2, 3, 1, 2, 3, and so on. Whether or not you can find a use for this is up to you!
