Technical Blog #2

March 29, 2014 by Kevin

Even though I have a bit of programming background from school, I never learned about hashes. I only learned how to use arrays and only discovered hashes when I started learning Ruby. I'll try to explain arrays and hashes and the uses for each of them in this blog post.


Arrays are like lists of objects. Imagine you have a grocery list of items you want to buy. If you were to find one thing in the list, you'd have to scan from top to bottom and look for it. If you wanted to be more organized, you can number each object and look for them more easily, so #4 of the list could be Milk. The 4 in terms of arrays is an index, while Milk is the value at that index.

Hashes function differently from arrays in that they're not really lists. Instead of each object in the list having a number assigned to it, hashes have key and value pairs. For example, hash["Basketball"] = "Sport" could be a pair, where Basketball is the key and Sport is the value.


Let's say we want to find a structure to store the ages of people in a city. In arrays, you would have to make the array multi-dimensional to store these pairs of data.

ages = [["Adam Smith", "33"], ["John Doe", "54"]]

To find the age of a particular person in our array, we'd have to start from the beginning and look through each element of the array. The code for that would look something like this:

ages.each {|x,y| puts y if x == "Adam Smith"}

Now that seems fine and dandy, but imagine if our ages array had hundreds and thousands of people in it, and Adam Smith were at the end. The longer the array is, the longer it will take to find the person.

If we were to use hashes instead, we can always use ages["Adam Smith"] to find the age of Adam Smith. That's much simpler than the array implementation and it'll take the same amount of time no matter how many people are stored in the hash. Because of how hashes work, the hash knows exactly where to go to retrieve the value when you give it the key.


Moving along, there are cool ways to use hashes and arrays together to do cool stuff. Here's an example of how I used a "histogram" to find the mode of an array for our bonus challenge this week:

mode_count = Hash.new(0)
array.each { |x| mode_count[x] += 1 }

This code will first make a new Hash and initialize the default value to be 0 for any key added. Then it iterates through an array and adds 1 to the value every time a key shows up in the array. I learned about this when I was first learning about hashes in Ruby and it blew my mind.


Overall, hashes are more efficient when it comes to performance compared to arrays. Even so, not everything is about efficiency though. Sometimes you just need the proper tool to fit the job. The more I try to differentiate the two, the more I realize that it's really easy in Ruby to do the same thing with either structure. Until I'm more experienced and have a better understanding, I'll say for now that arrays are useful for storing simple lists of data, while hashes are useful for storing data pairs that are easy to search for later.
