Cultural Blog #4

April 13, 2014 by Kevin

I used to be interested in mobile game development, and when it comes to small indie groups releasing monetized games, it is vital to protect cheating in order to keep up revenue. Frequently targeted values are gold, EXP, coins, etc. These values are critical so they must be protected in memory and on disk. I did some research in the past on how to deal with this problem, and here are my findings.

The two common ways of hacking games is save-game editing and memory editing.

Save-Game Editing

If the game saves are not encrypted, it is easy to access them and edit them inside a HEX editor to search for and change key values. It is easy to inspect the changes from one save to another if the data layout is consistent.

Once modified, the save can be downloaded on the internet as there is no check that the save belongs to a specific device.

Solutions:

  • Add Encryption: never write plain memory data. You can use XXTEA encryption for example.
  • Add check values: double/triple write your critical values with a different scheme and use this to verify that all values are coherent. You can write a short term history for these values for example.
  • Add a CRC check to verify that the data has not been modified externally.
  • Do not store these values inside a separate file but with other game data (map, etc.)

Memory Editing

Instead of modifying the save game, it is possible to scan the memory as the values are modified by the game. Once the memory locations of key values are identified, they can be patched in memory without the game noticing and they will then be saved as regular values for future sessions.

Solutions:

  • Make it harder to find the location of values. Do not store the value “as-is” in memory and make it move.
  • Detect that the value was not modified by the game itself. Keep a duplicate of your critical values in memory and verify them before any modification.

Of course, these solutions are mainly for games that don't require a connection to a server to play. When there is a server in the equation, these key values can be stored and modified on the server so cheating is almost impossible. But for smaller groups of developers that can't handle the bandwidth, these solutions are a relatively easy way of protecting your game.
