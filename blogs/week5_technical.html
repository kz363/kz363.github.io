Technical Blog #4

April 13, 2014 by Kevin

If you're new to Ruby, chances are that you've been working with classes and objects without knowing exactly what they are. Everything in Ruby is an object, but you can further subdivide them into a Class which share the same methods. You may have worked with some of these classes: String, Integer, Float, Array, Hash. Classes are always capitalized in Ruby convention, and are defined using the class keyword followed by the name. Here's an example:

class Card
 def initialize(value, suit)
  @value = value
  @suit = suit
 end
end

card = Card.new("A", "Spades")

Whenever you create a new instance of a class with .new, you can create new variables called instance variables using the @ operator. The instance variables in the code above are @value and @suit under the initialize method, which automatically runs when a new instance is created. In this example, they're equal to the strings "A" and "Spades" respectively. These instance variables can only be read or modified with that instance's methods. Currently there aren't any methods to do that, so let's write some!

def value
 return @value
end

def value=(value)
 @value = value
end

These two methods allow you to access @value using card.value. For example, you can print the value by doing puts card.value, or you can edit the value by doing card.value = 5. Both of the methods above are considered instance methods. Similar to instance variables, they only can only exist and be called from within an instance of a class.

Ruby actually has a convenient way of automatically generating methods to read and write to instance variables, so we don't need to write the code above for every variable.

class Card
 attr_accessor :value, :suit

 def initialize(value, suit)
  @value = value
  @suit = suit
 end
end

The code above allows you to write and read both @value and @suit without creating methods to do it for you. Of course, if you don't need to read and write, you can declare them with attr_reader or attr_writer depending on what you need. Isn't that neat?
